--- title: "Those mysterious ID3 URL frames" date: "2015-07-31T14:49:00+10:00" summary: "Four letter constants must be used." category: Media tag: - audio - guides - podcasting ---

To render (hah) podcasting less tedious, I've been writing a series of shell scripts to automate as much of the post-production as possible. The eyeD3 utility does a great job of inserting ID3 tags; throw it some strings, a local path for cover art and show notes (well, lyrics) and you're on your way.

This option caught my eye when reading its inline help:

--url-frame FID:URL   Set the value of a URL frame. To remove the frame,
                      specify an empty value. e.g., --url-frame='WCOM:'

On a whim, I tried setting a field with an FID called "WEBSITE".

Uncaught exception: Invalid URL frame-id

Well, then. Other than the example WCOM, the help didn't have a list of acceptable FIDs for this field. The ID3 website listed what they could be, but not the identifiers themselves.

It is also possible to include various links to webpages and other on line resources. These are the predefined URL frames':

  • Commercial information
  • Copyright/Legal information
  • Official audio file webpage
  • Official artist/performer webpage
  • Official audio source webpage
  • Official internet radio station homepage
  • Payment
  • Publishers official webpage
  • User defined URL link frame

As is so often the case, go to the source and get the answer. These are the defined constants:

URL_COMMERCIAL_FID = "WCOM"
URL_COPYRIGHT_FID  = "WCOP"
URL_AUDIOFILE_FID  = "WOAF"
URL_ARTIST_FID     = "WOAR"
URL_AUDIOSRC_FID   = "WOAS"
URL_INET_RADIO_FID = "WORS"
URL_PAYMENT_FID    = "WPAY"
URL_PUBLISHER_FID  = "WPUB"

With the exception of "User defined URL link frame", these line up with the predefined URL frames. So if you want to define a publisher:

eyeD3 $_mp3 --url-frame "WPUB:http://url.tld/"

I'm not sure which clients use these frames for anything, but I suppose it can't hurt setting them.
