There's a fascinating thread on Slashdot about what you'd include in your Frankenstein OS, assuming such a feat were possible. Below is mine.

NetBSD kernel
I'm a FreeBSD guy in daily use for pragmatic reasons, but NetBSD is achingly beautiful. Provided I could get drivers.

OpenBSD firewall
pf is the only firewall that doesn't make me want to remove hair.

Mac OS X font rendering
Windows fonts aren't great, and the average Linux desktop is awful.

Homebrew package manager
Keeping applications in their own trees is clean, easier to maintain and helps mitigate dependency hell.

BeOS desktop environment
It had elements of classic Mac OS, but it was even cleaner and simpler.

Windows 3.0 or 3.1 iconset
For the nostalgic feels.

OS/2 WarpCenter
Quick, simple glance at system status and application launching.

Yokogawa parallelism
If a CPU fails, I want to be able to pull it out and replace, live, and without skipping a beat.

Solaris ZFS
All other file systems are brittle and scary.

Vim keybindings
Everywhere. You can use a mouse and standard shortcuts, but still use Vim regex, navigation and visual editing in all the places.

Windows DirectX
I support OpenGL in principle, but I'd love to be able to play classic games without a compatibility layer. WINE is capable otherwise.

Perl shell with DOS or CP/M mode
Compatible with Bourne shell syntax (get those Bashims away from me), can run DOS and CP/M binaries unmodified, and accept DOS syntax when typed by accident (I grew up using it and still type cls constantly).

PalmOS mobile interface
For the mobile version of the OS. Still the gold standard in mobile UIs that no other platform has beat.

Complete ncurses interface
Based on Norton Commander, XTree Gold or whatever. Anything that can be done in the GUI and with text commands should have an nucures interface for easy, lightweight, remote use.
Built in Go
It's my flavour of the month, but other type safe languages would suffice.
