As American friends would lament posts like these as being a little too "Inside Baseball" (whatever that means), I've switched back to static site generation for Rubénerd, for the exact same reason I stopped doing so before.

Wait, what?

Like everyone reaching the end of their twenties, I'm realising the barrier to entry for my hobbies are time now, not technology. I'm still pigheaded enough to want to host my own material; as a tinkerer and "IT guy" I think it'll be a sad day when that's no longer true. So what can I do to blog with minimal effort, but with my own infrastructure?

For a couple of years, I thought Jekyll was the answer. Static generation drastically simplified my web server stack, and live previewing my site on my MacBook Air without internet was a boon. Ultimately though, 40 minutes to generate the site was a little silly, and I moved back to the enemy I know.

Enter Hugo

Hugo is written in Go, Google's alternative programming language. I've been tinkering with Go at work recently, and have been pleasently surprised (other than Generics, but that's another discussion).

The best thing about Hugo though is its fast. Compare the time to generate Rubénerd's 4000+ posts and metadata archives:

Engine Time
Jekyll, without tag archives 12:02
Jekyll, with jekyll-archives 42:04
Hugo, with archives 01:20

Hugo uses TOML, a twist on YAML that looks like a Windows INI file from the 1990s. Fortunately if you're not a fan, you can keep using the same frontmatter
