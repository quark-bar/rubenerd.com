--- title: "startx fails with .Xauthority not writable" date: "2015-07-29T14:16:00+10:00" summary: "As the error suggests, it’s an ownership issue" tag: - debian - guides - linux - turns-out - xorg ---

I was trying to start xorg on my token Linux tower this afternoon, but it kept timing out with errors. Among the last lines of output:

xinit: server error
xauth: /home/R/.Xauthority not writable, changes will be ignored

Turns out, I’d installed the packages with sudo from my home directory, then attempted to test X while still root. Therefore, the resulting Xauthority file was owned by root, so my regular user couldn't modify it. Derp.

-rw------- 1 root root 156 Jul 28 12:37 .Xauthority

I could have changed ownership, but I just deleted the file and ran startx again.
