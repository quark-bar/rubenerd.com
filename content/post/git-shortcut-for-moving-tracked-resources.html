--- title: "Git shortcut for moving tracked resources" date: "2015-05-12T09:19:00+10:00" summary: "Using the git rm command instead of rm and add" categories: - Software tags: - guides - git ---

Yes, I didn't know this until today, much like there's someone born every minute who hasn't seen The Flintstones. Previously to move a file, I would have done this:

$ mv ./original ./new
$ git rm ./original
$ git add ./new
$ git commit -m 'moved original to new'

Not to get all Malcolm Gladwell on you, but turns out, there's a mv command instead:

$ git mv ./original ./new
$ git commit -m 'moved original to new'
